## 🔐 Getting Your Credentials

### GitHub Token
1. Go to https://github.com/settings/tokens
2. Click "Generate new token"
3. Select scopes:
   - `repo` (full control)
   - `workflow` (GitHub Actions)
   - `admin:repo_hook` (webhooks)
4. Copy the token (starts with `ghp_`)

### Vercel Token
1. Go to https://vercel.com/account/tokens
2. Create new token
3. Copy token (starts with `VercelProdToken`)
4. Note your Team ID from https://vercel.com/account/settings

### Hetzner Server
- Need server IP/hostname
- Root SSH access (usually `root` user)
- Create SSH key: `ssh-keygen -t rsa -b 4096 -f ~/.ssh/hetzner_rsa`

### Option 3: Use Individual Classes
```javascript
const { GitHubAutoConnector, VercelAutoLinker, HetznerSSHManager } =

// Setup Vercel
const vercel = new VercelAutoLinker(token);

// Deploy to Hetzner
const hetzner = new HetznerSSHManager(host, user, keyPath);
await hetzner.connect();
await hetzner.deployRepository(repoUrl, '/root/unicorn-final');
await hetzner.disconnect();
```

## 📈 Deployment Flow

### Push Code to GitHub
```
git add .
git commit -m "Update feature"
git push origin main
```

### Automatic Deployment Chain
```
1. GitHub receives push
   ↓
2. GitHub triggers webhook → Hetzner (port 3001)
   ↓
3. Hetzner webhook server receives request
   ↓
4. Runs: cd /root/unicorn-final && git pull origin main
   ↓
5. Runs: npm install (if package-lock.json changed)
   ↓
6. Runs: npm run build
   ↓
7. PM2 auto-restarts application
   ↓
8. Meanwhile: Vercel also auto-deploys from GitHub
   ↓
✅ Both Vercel (frontend) and Hetzner (backend) updated!
```

## 🛡️ Security Features

✅ **GitHub Secrets** - Tokens stored encrypted in GitHub (not in code)
✅ **SSH Keys** - Private key on your machine, public key on server
✅ **Webhook Signature** - HMAC verification of GitHub webhooks
✅ **Environment Variables** - Sensitive data in `.env`, not in git
✅ **Admin Protection** - All admin routes use secret headers
✅ **No Password Login** - SSH key authentication only

## 📚 Module Classes

### GitHubAutoConnector
Methods:
- `createRepository()` - Create new repo
- `setupDeployKey(publicKey)` - Add SSH key
- `setupSecrets(secrets)` - Add GitHub Secrets
- `setupWebhook(url, secret)` - Configure webhook
- `setupWorkflow(yaml)` - Add GitHub Actions workflow

### VercelAutoLinker
Methods:
- `createProject(name, repo)` - Create Vercel project
- `setEnvironmentVariables(projectId, vars)` - Set env vars
- `triggerDeployment(projectId)` - Start deployment
- `getDeploymentStatus(id)` - Check deployment status

### HetznerSSHManager
Methods:
- `connect()` - Establish SSH connection
- `deployRepository(url, path)` - Clone and build repo
- `setupWebhookServer(path, secret)` - Deploy webhook listener
- `setupAuthorizedKeys(publicKey)` - Add SSH key
- `disconnect()` - Close SSH connection

### PlatformOrchestrator
Methods:
- `setupAll(config)` - Run complete setup

```bash

openssl rand -hex 32

# Update in .env.auto-connector
GITHUB_SECRET_ADMIN=new-secret-here

# Run setup again (it updates secrets)
bash setup-platform-auto-connect.sh

# OR update manually in GitHub Settings → Secrets
# and on Hetzner in .env file
```

### Example 3: Scale to Multiple Hetzner Servers
```javascript
const connector = require('./github-vercel-hetzner-connector.js');

const servers = [
  { host: 'server1.example.com', path: '/root/unicorn-1' },
  { host: 'server2.example.com', path: '/root/unicorn-2' },
  { host: 'server3.example.com', path: '/root/unicorn-3' }
];

for (const server of servers) {
  const hetzner = new HetznerSSHManager(
    server.host,
    'root',
    '~/.ssh/hetzner_rsa'
  );
  await hetzner.connect();
  await hetzner.deployRepository(repoUrl, server.path);
  await hetzner.disconnect();
}
```
